它不是只给你一批关键词，而是把关键词调研、流量分层、广告结构建议、否词整理、Listing 联动整理成一套可执行结果。

默认交付内容：
- 关键词总库
- 否定关键词库
- 旺季备战看板

推荐补齐的核心字段：
- 流量层级
- 词根类型
- 精准度
- 匹配类型建议
- 广告策略建议
- Listing 联动位置

## 适用人群

这个仓库适合下面几类人直接使用：
- 想给新品先搭一套可投放的关键词库
- 想重做老链接关键词结构
- 想判断关键词该不该投、该怎么投、要不要回写 Listing
- 想把关键词研究结果整理成飞书多维表格
- 想按固定 SOP 检查词库结果有没有退回旧逻辑

## 你会得到什么

这个 skill 重点解决的是“关键词很多，但不会落地”的问题。

它会帮助你补齐这些判断：
- 哪些词是大词、中词、长尾词、品牌词
- 哪些词适合广泛，哪些词适合精准守位
- 哪些词应该进入否词库
- 哪些词适合回写标题、卖点、A+
- 旺季前应该优先补哪些词

## 快速开始

如果你只想先跑通一次，直接按下面做：

### 第一步：下载仓库

如果你会用 Git：

```bash
git clone https://github.com/luotwo/BSC-SIF-Amazon-keyword-database/tree/main
cd BSC-SIF-Amazon-keyword-database
```

如果你是新手：
- 在 GitHub 页面点击 `Code`
- 选择 `Download ZIP`
- 解压后，用 Claude Code 打开这个目录

### 第二步：准备 Claude 环境

你至少需要：
- 一个可用的 Claude 环境，推荐 `Claude Code`
- 已启用 MCP
- 可访问 SIF MCP

如果你还要输出飞书多维表格，还需要：
- 可用的飞书能力
- 对应的 `lark-base` 能力已可调用

### 第三步：配置 MCP

仓库里自带示例文件：`mcp-config.json`

当前示例配置：

```json
{
  "mcpServers": {
    "sif-amazon": {
      "url": "https://mcp.sif.com/mcp?secret-key=YOUR_SIF_SECRET_KEY"
    }
  }
}
```

使用方式：
- 如果你的 Claude 环境支持直接加载 MCP 配置，直接复用这个文件
- 如果你已经有自己的 MCP 配置，把 `sif-amazon` 这一段合并进去
- 如果你的环境已经能正常调用 SIF MCP，这一步跳过即可

### 第四步：第一次提问直接这样说

```text
请基于这个 skill，帮我搭建 US 站某产品的关键词库。
要求：
- 按 48+ 竞品 ASIN 口径做主力拓词
- 输出关键词总库、否词库、旺季备战看板
- 每个关键词尽量带出流量层级、词根类型、精准度、匹配类型建议、Listing 联动位置
- 不要脑补，没有数据支持时不要输出精确预算
```

如果你只想检查现有结果，也可以这样问：

```text
请按当前 SKILL.md 和 BENCHMARK.md 的口径，检查这版关键词库有没有退回旧逻辑。
重点检查：
- ABA 分层是否正确
- 是否把机会词误当固定层级
- 否词逻辑是否过度
- 广告打法是否符合当前 SOP
- 预算建议是否退回平均分配
```

如果你要输出飞书版：

```text
请把这个关键词库整理成飞书多维表格交付版。
要求：
- 输出关键词总库、否词库、旺季备战看板
- 保留广告结构标签
- 没有可验证数据时，建议出价和建议日预算留空
- 不要把机会词写成固定层级字段
```

## 第一次使用建议顺序

如果你是第一次接触这个仓库，推荐按下面顺序走：
1. 先看 `README.md`
2. 再看 `SKILL.md`
3. 再看 `BENCHMARK.md`
4. 选定站点和品类
5. 准备 `48+` 竞品 ASIN
6. 跑出关键词总库
7. 补否词库和旺季看板
8. 最后按 `BENCHMARK.md` 做回归验收

## 输出结果长什么样

一次合格输出，至少应该包含下面三部分。

### 1. 关键词总库
建议至少包含这些字段中的大部分：
- 关键词
- 流量层级
- 词根类型
- 精准度
- 匹配类型建议
- 广告策略建议
- Listing 联动位置

### 2. 否定关键词库
建议至少包含：
- 否词
- 否词原因
- 不相关或不值得投放的理由

### 3. 旺季备战看板
至少能回答：
- 哪些词要提前布局
- 哪些词属于核心活动优先级
- 哪些词属于增长活动优先级
- 哪些词适合测试，不适合重预算

## 新手先记住这几条

这是当前版本最重要的使用原则：
- 主力拓词优先基于 `48+` 竞品 ASIN，不是只看 `5-10` 个样本
- 流量层级优先用真实 `ABA排名`，不要只按搜索量硬分层
- `机会词` 只能作为投放优先级标签，不能当作固定层级
- 品牌词、受众词、属性长尾词要分开处理
- 否词库不能空，但也不能把一般精准词一刀切全否掉
- 预算不能平均分给所有活动
- 没有可验证的 `CPC / CVR / ACOS` 数据时，不输出伪精确预算和出价

## Source-faithful 边界

这个仓库有一个硬规则：只写知识库或当前数据直接支持的结论，不脑补。

也就是说：
- 可以输出结构化判断，比如 `流量层级`、`词根类型`、`精准度`
- 可以输出广告结构标签，比如 `SP-Core-Broad`
- 不可以把知识库没明确支持的经验硬写成结论
- 不可以在缺少数据时伪造精确 `建议出价`、`建议日预算`
- 不可以把 `机会词` 当成固定层级

## 常见误区

这个仓库明确反对这些做法：
- 只给一批关键词，不给结构字段
- 只看搜索量，不看 `ABA排名`
- 把 `机会词` 当成固定层级
- 否词库空表
- 把品牌词和普通词混成一套投放逻辑
- 所有词都写成同一种 Listing 联动位置
- 预算平均分给所有活动
- 在没有数据时写看起来很专业的精确出价和预算

## 如何判断结果合不合格

先看 `BENCHMARK.md`。

当前最低通过标准是：
- 关键词总库不少于 `20` 条
- 否词库不少于 `3` 条
- 旺季备战看板与总库主集合基本对齐
- 总库无重复关键词
- 至少抽检 `5` 个关键词，确认 `ABA排名 → 流量层级` 映射正确
- 至少抽检 `5` 个关键词，确认 `词根类型 + 精准度 + 匹配类型建议` 合理
- 至少存在 `标题前部 / 卖点首行 / A+文案 / 不回写` 四种 Listing 联动结果
- 不把 `机会词` 当固定层级
- 预算建议不采用平均分配口径

## 常见问题

### 1. 这个仓库是给谁用的？

给会在 Claude 环境里跑亚马逊关键词研究、词库整理、飞书交付的人用。

### 2. 我不会 Git，可以用吗？

可以。直接在 GitHub 下载 ZIP，解压后用 Claude Code 打开目录即可。

### 3. 我没有飞书能力，可以先用吗？

可以。你可以先只跑关键词总库和判断逻辑，不落飞书表。

### 4. 没有真实广告数据时能不能给精确预算？

不能。当前规则是不输出没有数据支持的伪精确预算和出价。

### 5. 我应该先看哪个文件？

第一次使用建议顺序：`README.md` → `SKILL.md` → `BENCHMARK.md`
