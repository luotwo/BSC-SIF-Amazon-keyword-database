# 高级亚马逊运营 SOP：亚马逊关键词库搭建

本技能集成了 **SIF MCP 工具**、**lark-base 飞书多维表格 skill** 与 **llm-wiki 知识库实战经验**。

> 所有判断逻辑均来自知识库实战页面，不做脑补。依据来源：`amazon-keyword-library-management`、`amazon-advertising-architecture`、`amazon-keywords-and-traffic-diagnosis`、`amazon-big-keyword-vs-long-tail-strategy`、`amazon-auto-search-term-to-manual-migration`、`amazon-ad-five-adjustment-directions`、`amazon-ad-budget-scaling`、`amazon-keyword-relevance-screening`、`amazon-keyword-position-tracking`。

## Source-faithful 边界

- 只能输出 **知识库页面明确支持** 或 **当前 SIF 数据直接支持** 的判断。
- 允许输出结构性结果：`流量层级`、`词根类型`、`精准度`、`匹配类型建议`、`Listing联动`、`广告结构标签`。
- `机会词` 只能作为投放优先级标签，不能当作固定层级。
- 若当前执行任务没有可验证的 `CPC / CVR / ACOS` 数据，不得把预算公式硬写成精确数值结果。
- 默认情况下，飞书中的 `建议出价`、`建议日预算` 这类精确数值字段应留空；除非用户明确提供目标，或当前任务已经拿到足够可验证的数据链路。
- `广告组名称` 可以作为执行组织标签写入，但它是结构化命名，不是知识库原文逐条结论。

## 数据完整性分级

执行前明确哪些字段是必填、哪些是降级处理：

| 字段来源 | 级别 | 当数据不可得时 |
|---------|------|--------------|
| SIF 周均搜索量 | **必填** | 无数据则不入库 |
| SIF 历史趋势 / YoY | **必填** | 无数据则不入库 |
| ABA 真实排名 | **推荐** | 以 SIF 周均搜索量换算估算，字段标注 `[估算]` |
| SIF 竞争强度 | **推荐** | API 不可用时留空，字段标注 `[缺失]` |
| 共性筛选占位率 | **推荐** | 低于 48 ASIN 时标注置信度低 |

**不允许**：用估算值填写必填字段，或脑补知识库不支持的判断。

## 核心流程

### 第一步：多竞品拓词（SIF MCP）

> 知识库原文：SIF 系统要求最少选择 **48 个竞品 ASIN**，低于此数占位率不准，也会影响词相关性的高低。5-10 个竞品反查只是起始补充，不是主要来源。

#### 1.1 主力拓词路径

```
# 对每一个竞品 ASIN 调用（最少 48 个）
sif: market_get_asin_keyword_signals(asin=竞品ASIN, country="US")

# 聚合所有 ASIN 的词信号
# 按"出现在多少个 ASIN"排序 → 筛选共性关键词（≥ 14 个 ASIN 包含）
```

#### 1.2 补充拓词路径（在主力拓词后补充）
- 以词拓词：`market_get_keyword_competition(keyword=核心词)` 找相关词
- 历史趋势拓词：`market_get_keyword_root_trend(keyword=词根)` 找同词根下的长尾词
- 批量需求筛查：`market_get_keyword_demand(keywords=[候选词列表])` 验证搜索量

#### 1.3 词量目标
- 多竞品拓词原始词库：目标 500-2000 词
- 经过共性筛选后的核心词库：100-300 词
- 最终投放词库（过滤低精准/低相关）：50-150 词

---

### 第二步：关键词四层分层管理

知识库明确给出四个维度，按顺序处理，不跳步。

#### 第一层：共性关键词筛选
- 筛选"相关产品数"≥ 14 的词（大多数竞品 ASIN 都包含的词）。
- 共性关键词是产品**最常规、最常用的叫法**，不只看能否搜到，更看是否是行业通用表达。
- 同一词的单复数只保留搜索量更大的一个。
- 搜索量过低的词（周均 < 50）暂不入主库，可入长尾补充库。

#### 第二层：词根分类
按词频统计表找出高频词根，分为六类：

| 词根类型 | 判断标准 | 广告用途 |
|---------|---------|---------|
| **共性词根** | 产品最基础叫法，绝大多数 ASIN 包含 | 必须覆盖，广泛/词组/精准分层投放 |
| **属性词根** | 修饰产品规格/特性，能与共性词组成精准词组 | 精准投放，小词带大词策略核心词根 |
| **品牌词根** | 竞品品牌名 | 精准截流，独立广告组 |
| **品类词根** | 同类产品的其他叫法 | 广泛覆盖，精准度按前台结果判断 |
| **受众词根** | 描述用户身份/场景的词根 | 精准首选，配合 SB 人群定向 |
| **否定词根** | 前台搜索结果不精准、转化率低，或能与共性词组成词组但结果不相关 | **直接进否词库，不入主库** |

#### 第三层：按 ABA 排名划分流量量级

> **数据来源说明**：优先使用真实 ABA 排名数据。当 ABA 数据不可得时，以 SIF 周均搜索量按以下换算估算，并在"流量层级"字段标注 `[估算]`。

| 层级 | ABA 排名区间 | 周均搜索量换算 | 广告策略含义 |
|------|------------|--------------|------------|
| 大词 | < 10 万 | 周均 > 12,500 | 高竞争，新品 0 评论期不打首页首位精准，先用长尾词带动 |
| 中词 | 10-50 万 | 周均 2,500-12,500 | 主推窗口，手动广泛先行建立权重 |
| 长尾词 | 50-100 万 | 周均 500-2,500 | 精准匹配，转化稳定后带动中大词排名 |
| 小词 | > 100 万 | 周均 < 500 | 精准首选，用于 Listing 埋词和补充流量入口 |
| 品牌词 | 不适用 | 不限 | 独立预算，精准匹配截流，不与普通词混投 |

#### 第四层：精准度分类

前台搜索结果验证，判断买家意图匹配程度：

| 精准度 | 判断标准 | 处理方式 |
|-------|---------|---------|
| **绝对精准** | 买家搜此词时，目标基本就是这类产品 | 主力投放词，按层级策略操作 |
| **相对精准** | 买家意图有一定匹配，但结果也包含其他类目 | 可投，需监控 CTR 和 CVR |
| **一般精准** | 词义宽泛，前台出现较多其他类目产品 | 谨慎纳入，优先考虑归入否词候选 |

#### 关于“机会词”的处理口径

- 知识库未把“机会词”定义为固定层级。
- 仅当某词同时满足 **高相关、具备一定搜索量、竞争强度可承受、前台验证可投** 时，才可在“广告策略建议”中标记为机会切入词。
- “机会词”是投放优先级标签，不替代 `流量层级 / 词根类型 / 精准度` 三个基础字段。
- 禁止把“机会词”单独当作流量层级写入飞书字段，避免退回旧口径。

---

### 第三步：竞争强度评估（SIF MCP 补充层）

> **说明**：竞争强度是附加维度，影响出价策略和预算分配优先级。当 API 不可用时字段留空，不影响其他层级判断。

```
# 对每个关键词查询竞争情况
sif: market_get_keyword_competition(keyword=目标词, marketplace="US")
```

从返回结果提取：
- **CPC 参考**：当前该词的广告竞价区间
- **竞争程度**：头部 ASIN 占据的流量份额集中度
- **竞品排名变化**：近期竞品是否在该词上发力

根据竞争强度修正匹配类型建议：
- 高竞争大词 + 新品 → 强制锁定为"广泛先行"，不允许填"精准首选"
- 低竞争长尾词 + 绝对精准 → 可直接"精准首选"，CPC 成本低

---

### 第四步：广告投放与控制

> 知识库实战：**新品不要一上来打大词精准**，除非利润空间足够高或站点竞争很小。正确路径是先广泛建立数据，再让数据决定什么时候上精准。

#### 广告打法顺序（知识库依据）
1. **自动广告先跑**：承担收录、探词、跑初始转化数据的任务。前期不要过早大量否词，避免封死流量入口。
2. **手动广泛先行**：CPC 比精准便宜，可推关键词自然位和广告位，先建立权重。
3. **小词带大词**：先推包含大词的长尾词排名（如 ABCD 词组），长尾词上了第一页后，大词自然排名也会跟着提升，再开大词精准时 CPC 更低、转化更高。
4. **数据驱动转精准**：当某词出了多次稳定订单，说明它是真实转化词，此时再开精准匹配卡位。

#### 新品 0 评论的特殊约束
- 不要上首页首位打大词精准 → 会沦为炮灰，ACOS 爆表。
- 先测其余位置和商品页面，等评论积累后再攻首页。

#### 预算计算方式（知识库依据）
```
每日预算 = 目标单量 × CPC ÷ CVR
```
- 该公式仅用于单活动起始预算估算，不代表整盘预算平均分配。
- 总预算应优先分给三类活动：`核心活动`、`增长活动`、`辅助/测试活动`，禁止所有活动平均分预算。
- `核心活动` 优先保障品牌防御、核心词、已验证盈利活动；`增长活动` 承担放量；`辅助/测试活动` 仅保留受控测试预算。
- 广告表现好时加预算要克制：先加 1-2 美金观察，不要粗暴翻倍。
- ACOS 低于目标时可增加预算 20%，高于目标时减少 10%。
- 预算上限需受盈亏平衡 ACOS 约束；新品测试期可短暂高于平衡点，进入稳定期后应回到平衡点附近。
- 当自然订单占比已经较高时，应优先评估是否释放预算保利润，而不是机械继续扩量。

#### 匹配类型建议判断矩阵

| 流量层级 | 词根类型 | 精准度 | 匹配类型建议 |
|---------|---------|-------|------------|
| 大词 / 中词 | 共性词根 / 品类词根 | 绝对/相对精准 | 广泛先行 |
| 大词 / 中词 | 共性词根 | 绝对精准（已有转化数据） | 词组过渡 → 精准守位 |
| 长尾词 | 属性词根 | 绝对精准 | 精准守位（小词带大词） |
| 小词 | 受众词根 / 属性词根 | 绝对精准 | 精准首选 |
| 品牌词 | 品牌词根 | 绝对精准 | 精准首选（独立预算）|
| 任何层级 | 任何词根 | 一般精准 | 否词候选，不投放 |

### 第五步：否词库建立

> 知识库原文强调：**不要过早大量否词**，避免把流量入口封死。

#### 否词三批来源
1. **词频统计里明显不相关和不精准的词根**（拓词阶段即可处理）
2. **前台搜集到的同行品牌词**（品牌词根归入竞品截流组，不做普通否词）
3. **广告报告里跑出来的不相关词**（投放后数据驱动）

#### 日常否词触发条件
- 点击 5-10 次仍不出单，**且**前台相关性低
- 不满足相关性条件时，即使不出单也不要急于否定（可能是出价/图片问题）

---

### 第六步：精准词 → Listing 联动

> 知识库原文：跑出来的高效精准词，不只是继续留在广告活动里，也可以同步回写到 Listing。

#### 触发条件
- 某词出现 3 次以上稳定转化订单 → 进入"Listing 联动候选"

#### 回写优先级
| 优先级 | 词类型 | 回写位置 |
|-------|-------|---------|
| P1 | 高效转化共性词 / 属性词 | 标题前部词位 |
| P2 | 高效属性词 | 五条卖点首行 |
| P3 | 品类词 / 受众词 | A+ 文案 / Search Terms 后台 |
| 不回写 | 否词、品牌截流词 | — |

---

### 第七步：流量诊断逻辑（SIF 流量时光机）

按以下顺序排查，不要跳步：

```
sif: ops_get_asin_traffic_trend(asin=目标ASIN, granularity="week")
```

1. 先看**市场大盘**：确认是行业整体问题还是自己链接问题
2. **定位异常日期**：切到按天（granularity="day"），找到流量突然下跌的具体日期
3. **拆流量类型**：自然流量（nfScore）vs SP/SB/SBV 广告各自跌幅
4. **锁定关键词根因**：调用 `ops_get_asin_traffic_trend_detail` 找出具体掉流量的核心词
5. 判断是自然排名掉、广告位掉还是竞品抢位

## 执行顺序与 SIF 工具调用清单

```
# 阶段一：拓词（必须执行）
FOR each competitor_asin IN asins[48+]:
    market_get_asin_keyword_signals(asin=competitor_asin, country="US")

# 阶段二：需求验证（必须执行）
market_get_keyword_demand(keywords=[候选词列表], marketplace="US")

# 阶段三：历史趋势（必须执行）
FOR each keyword IN 候选词库:
    market_get_keyword_history(keywords=[keyword], marketplace="US")

# 阶段四：竞争强度（推荐执行，不可用时跳过标注缺失）
FOR each keyword IN 核心词库:
    market_get_keyword_competition(keyword=keyword, marketplace="US")

# 阶段五：词根趋势补充（可选，用于长尾词挖掘）
FOR each root IN 高频词根:
    market_get_keyword_root_trend(keyword=root, marketplace="US")
```
